# Approved Samples

This folder is the agent's memory. Every file here is content you've approved. The agent loads 2 random samples each run as live examples of correct LIC output.

## How to add a sample

When the agent produces something you like:
1. Copy the content into a new .md file in this folder
2. Name it: keyword-slug-type.md (e.g. full-spectrum-thc-benefits-blog.md)
3. Add the header block at the top (copy from existing file)
4. Commit and push

## Header format

```
# APPROVED SAMPLE
# Keyword: [keyword]
# Date approved: [YYYY-MM-DD]
# Type: blog_post | product_copy
# Notes: [what worked, what to replicate]
```

## What makes a good sample?
- Sounds unmistakably like LIC
- CTA links to a specific product
- Hook is experiential, not clinical
- You'd publish it with zero edits

The more samples here, the better the agent's output gets over time.
